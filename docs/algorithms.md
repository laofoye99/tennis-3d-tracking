# Tennis 3D Ball Tracking System - 算法与计算逻辑文档

## 1. 处理流水线概览

```
原始帧 (1920×1080 BGR)
  │
  ├─[1] 预处理 ──→ 归一化张量 (3, 288, 512)
  │
  ├─[2] 时序堆叠 ──→ 输入张量 (1, 9, 288, 512)
  │
  ├─[3] ONNX 推理 ──→ 原始输出 (1, 3, 288, 512)
  │
  ├─[4] Sigmoid ──→ 概率热力图 (3, 288, 512)    值域 [0, 1]
  │
  ├─[5] 后处理 ──→ 像素坐标 (pixel_x, pixel_y)
  │
  ├─[6] 单应性变换 ──→ 世界坐标 (world_x, world_y)  米
  │
  └─[7] 三角测量 ──→ 3D 坐标 (x, y, z)  米
```

## 2. 图像预处理

### 2.1 步骤

```python
# 输入: frame (1920×1080, BGR, uint8)
img = cv2.resize(frame, (512, 288))          # 1. 缩放到模型输入尺寸
img = cv2.cvtColor(img, cv2.COLOR_BGR2RGB)    # 2. BGR → RGB
img = img.astype(np.float32) / 255.0          # 3. [0,255] → [0,1]
img = (img - MEAN) / STD                      # 4. ImageNet 归一化
img = img.transpose(2, 0, 1)                  # 5. HWC → CHW
```

### 2.2 ImageNet 归一化参数

```
MEAN = [0.485, 0.456, 0.406]    # R, G, B 通道均值
STD  = [0.229, 0.224, 0.225]    # R, G, B 通道标准差
```

### 2.3 时序堆叠

模型需要连续 3 帧作为上下文:

```
Frame t-2: (3, 288, 512)  ─┐
Frame t-1: (3, 288, 512)  ─┼─ concatenate(axis=0) → (9, 288, 512)
Frame t:   (3, 288, 512)  ─┘                         ↓
                                        unsqueeze(0) → (1, 9, 288, 512)
```

通道排列: `[R₀G₀B₀ R₁G₁B₁ R₂G₂B₂]`，共 9 个通道。

## 3. ONNX 模型推理

### 3.1 模型规格

| 属性 | 值 |
|------|-----|
| 模型 | HRNet (High-Resolution Network) |
| 文件 | `model_weight/hrnet_tennis.onnx` (~5MB) |
| 输入名 | `input` |
| 输入形状 | `[batch, 9, 288, 512]` |
| 输出名 | `output` |
| 输出形状 | `[batch, 3, 288, 512]` |
| 任务 | 网球热力图回归 |

### 3.2 GPU 加速 (IOBinding)

```python
# 将输入直接放在 GPU 显存中，避免 CPU↔GPU 拷贝开销
input_torch = torch.from_numpy(input_tensor).cuda()
io_binding.bind_input(
    name="input",
    device_type="cuda", device_id=0,
    element_type=np.float32,
    shape=tuple(input_torch.shape),
    buffer_ptr=input_torch.data_ptr()    # 直接传 GPU 指针
)
io_binding.bind_output(name="output")
session.run_with_iobinding(io_binding)
output = io_binding.copy_outputs_to_cpu()[0]    # 仅拷贝输出
```

### 3.3 后激活

模型原始输出为 logits，需要 sigmoid 转为概率:

```python
output = sigmoid(raw_output)    # 值域 [0, 1]
# output.shape = (3, 288, 512)
# 每一帧 (288×512) 是一张热力图，高值区域 = 网球可能位置
```

## 4. 热力图后处理

### 4.1 流程

```
热力图 (288×512)
    │
    ├─ resize → (1920×1080)          匹配原始分辨率
    │
    ├─ 阈值过滤 (> 0.5)              去除低置信度噪点
    │
    ├─ 连通域分析                     分离多个候选区域
    │      connectedComponentsWithStats()
    │
    ├─ 加权质心计算                   每个 blob 的精确中心
    │      cx = Σ(x·h) / Σ(h)
    │      cy = Σ(y·h) / Σ(h)
    │
    └─ 最优 blob 选择                 追踪逻辑
           ├─ 有历史 → 选最近预测位置的 blob
           └─ 无历史 → 选置信度最高的 blob
```

### 4.2 加权质心公式

对每个连通域 blob:

```
         Σ  x_i · h(x_i, y_i)
cx = ─────────────────────────
         Σ  h(x_i, y_i)

         Σ  y_i · h(x_i, y_i)
cy = ─────────────────────────
         Σ  h(x_i, y_i)
```

其中 `h(x, y)` 是该像素的热力图值，`(x_i, y_i)` 是 blob 内所有像素坐标。

confidence = `Σ h(x_i, y_i)` (blob 内热力图值总和)

### 4.3 运动预测追踪

维护最近 3 帧的检测位置历史 `prev_positions`:

```python
# 线性外推预测下一帧位置
velocity = prev_positions[-1] - prev_positions[-2]
predicted = prev_positions[-1] + velocity

# 选择距离预测位置最近的 blob
distances = [euclidean(blob_center, predicted) for blob in blobs]
best_blob = blobs[argmin(distances)]
```

目的: 当画面中有多个高亮区域(如球拍反光)时，利用运动一致性选择真正的网球。

## 5. 单应性变换 (Homography)

### 5.1 原理

单应性矩阵 H 描述了同一平面在两个视角之间的投影关系。
这里将图像像素平面映射到球场地面平面 (z=0 的世界坐标):

```
┌     ┐     ┌             ┐   ┌     ┐
│ w·X │     │ h11 h12 h13 │   │ px  │
│ w·Y │  =  │ h21 h22 h23 │ × │ py  │
│  w  │     │ h31 h32 h33 │   │  1  │
└     ┘     └             ┘   └     ┘

世界坐标: X = (w·X) / w,  Y = (w·Y) / w
```

### 5.2 标定过程

使用 12 个球场关键点进行标定:

```
球场关键点标注 (Labelme JSON):
  ├── left_top           (0, 23.77)      远端底线左角
  ├── left_top_serve     (0, 18.285)     远发球线左角
  ├── left_bottom_serve  (0, 5.485)      近发球线左角
  ├── left_bottom        (0, 0)          近端底线左角
  ├── center_top         (4.115, 23.77)  远端中线
  ├── center_top_serve   (4.115, 18.285) 远发球线中点
  ├── center_bottom_serve(4.115, 5.485)  近发球线中点
  ├── center_bottom      (4.115, 0)      近端中线
  ├── right_top          (8.23, 23.77)   远端底线右角
  ├── right_top_serve    (8.23, 18.285)  远发球线右角
  ├── right_bottom_serve (8.23, 5.485)   近发球线右角
  └── right_bottom       (8.23, 0)       近端底线右角
```

### 5.3 相对视角映射

两个相机从对面看球场，相同标签名对应不同物理位置:

```
Camera 66 (从 y=0 端看):
  "left_bottom" = 近端底线左角 = 世界坐标 (0, 0)
  "left_top"    = 远端底线左角 = 世界坐标 (0, 23.77)

Camera 68 (从 y=23.77 端看):
  "left_bottom" = 近端底线左角 = 世界坐标 (8.23, 23.77)  ← 物理上是另一端
  "left_top"    = 远端底线左角 = 世界坐标 (8.23, 0)

cam68 的 "left" = cam66 的 "right" (x 轴翻转)
cam68 的 "bottom" = cam66 的 "top" (y 轴翻转)
```

### 5.4 计算方法

```python
H, mask = cv2.findHomography(src_pixels, dst_world, cv2.RANSAC, 5.0)
```

- 使用 RANSAC 鲁棒估计，容差 5.0 像素
- 12 个点超定，最少需要 4 个点
- RANSAC 自动排除异常点

### 5.5 精度

| 相机 | 使用点数 | 平均重投影误差 |
|------|---------|--------------|
| Camera 66 | 12 | 0.053 m (5.3 cm) |
| Camera 68 | 12 | 0.030 m (3.0 cm) |

### 5.6 使用

```python
# 像素 → 世界 (用于检测后的坐标转换)
pt = H_img2world @ [pixel_x, pixel_y, 1]
world_x = pt[0] / pt[2]
world_y = pt[1] / pt[2]

# 世界 → 像素 (用于可视化叠加)
pt = H_world2img @ [world_x, world_y, 1]
pixel_x = pt[0] / pt[2]
pixel_y = pt[1] / pt[2]
```

**注意**: 单应性变换假设目标在地面平面 (z=0)。球在空中时，像素→世界的投影是球到地面的
"影子"位置，不是球的实际 xy 位置。这就是为什么需要两个视角的三角测量来恢复真实 3D 位置。

## 6. 3D 三角测量 (Triangulation)

### 6.1 问题描述

已知:
- 相机 1 的 3D 位置 `C₁ = (cx₁, cy₁, cz₁)`
- 相机 2 的 3D 位置 `C₂ = (cx₂, cy₂, cz₂)`
- 相机 1 看到球投影在地面的世界坐标 `G₁ = (wx₁, wy₁, 0)`
- 相机 2 看到球投影在地面的世界坐标 `G₂ = (wx₂, wy₂, 0)`

求: 球的 3D 位置 `P = (x, y, z)`

### 6.2 射线模型

```
          C₁ (相机1)                          C₂ (相机2)
          /                                        \
         /  射线 r₁                    射线 r₂       \
        /                                            \
       /    P (球)●─ ─ ─ ─ ─ ─ ─ ─ ─●P'              \
      /        /                        \              \
     /        / 最近距离                  \              \
    ▼        ▼                            ▼              ▼
   G₁ (地面投影)                         G₂ (地面投影)
═══════════════════ 地面 z=0 ═══════════════════════════
```

每条射线从相机位置出发，穿过地面投影点:

```
r₁(s) = C₁ + s · (G₁ - C₁),   s ∈ [0, 1]
r₂(t) = C₂ + t · (G₂ - C₂),   t ∈ [0, 1]
```

当 `s=0` 时在相机位置，`s=1` 时在地面投影点。

### 6.3 优化求解

两条射线在 3D 空间中通常不会精确相交（异面直线），
需要找到两条射线上距离最近的一对点:

```
minimize   ‖r₁(s) - r₂(t)‖₂
   s,t

subject to:
   0 ≤ s ≤ 1
   0 ≤ t ≤ 1
   z₁(s) ≥ 0    (球在地面以上)
   z₂(t) ≥ 0
```

使用 `scipy.optimize.minimize` 求解。

### 6.4 3D 坐标计算

```python
# 最优参数
s_opt, t_opt = result.x

# 两条射线上的最近点
P₁ = C₁ + s_opt · (G₁ - C₁)
P₂ = C₂ + t_opt · (G₂ - C₂)

# 取中点作为球的 3D 位置
P = (P₁ + P₂) / 2
```

### 6.5 几何直觉

```
s = 0.0:  点在相机处     (高处)
s = 0.5:  点在射线中间    (球在空中)
s = 1.0:  点在地面投影处  (球在地面)

如果两个相机都观察到球在同一 xy 位置:
  → 两条射线交于地面 → z ≈ 0 (球在地面)

如果两个相机观察到球在不同 xy 位置 (视差):
  → 两条射线在空中最近 → z > 0 (球在空中)
  → 视差越大，z 越高
```

### 6.6 时间同步

两个相机的检测结果通过时间戳配对:

```python
MATCH_WINDOW = 0.1  # 秒

dt = abs(cam66_detection.timestamp - cam68_detection.timestamp)
if dt < MATCH_WINDOW:
    # 配对成功，执行三角测量
    triangulate(...)
```

100ms 的窗口在 30fps 摄像头下约为 3 帧的容差。

## 8. 精度分析与局限性

### 8.1 误差来源

| 环节 | 误差来源 | 估计量级 |
|------|---------|---------|
| 标定 | 手动标注关键点精度 | ~3-5 像素 |
| 单应性 | 重投影误差 | 3-5 cm |
| 检测 | 热力图分辨率 (288×512 → 1920×1080) | ~5 像素 |
| 检测 | blob 质心精度 | ~2-3 像素 |
| 三角测量 | 时间同步误差 (100ms 窗口) | 取决于球速 |
| 三角测量 | 单应性 z=0 假设 | 球越高偏差越大 |

### 8.2 关键假设

1. **地面平面假设**: 单应性变换假设目标在 z=0 平面上。球在空中时，
   像素→世界的投影实际上是球到地面的"伪投影"，不是真实 xy 位置。
   这由三角测量通过两个视角的视差来补偿。

2. **相机固定假设**: 相机位置和姿态不变，预计算的单应性矩阵始终有效。
   如果相机发生物理位移需要重新标定。

3. **单球假设**: 追踪逻辑假设场上只有一个网球。多球情况下需要更复杂的
   数据关联算法。

### 8.3 视差与高度精度

球的高度 z 完全依赖两个视角之间的视差:

```
视差 = 0     → z ≈ 0 (球在地面)
视差增大     → z 增大 (球在空中)
视差 = 最大  → z = 相机高度 (理论上限)
```

两个相机都在球场同一中线 (x=4.115m) 的两端，主要视差在 y 方向。
对于 y 坐标接近球网的球，两个相机的 y 投影差异最大，高度估计最准确。
对于接近某一端底线的球，该端相机几乎正视，视差较小，高度精度降低。
