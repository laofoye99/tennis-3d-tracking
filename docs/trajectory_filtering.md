# 轨迹滤波方法选型与设计原则

## 1. 核心原则：检测与滤波必须解耦

### 1.1 问题背景

在本项目中，我们曾使用 `BallTracker` 维护历史位置队列，通过线性外推预测下一帧球的位置，
并用预测位置来辅助选择 blob。实测发现这种方式会产生**级联错误**：

```
帧 63: 预测位置偏移 → 选了错误的 blob
帧 64: 错误 blob 进入历史 → 预测进一步偏移
帧 70-74: 错误持续累积，完全偏离真实轨迹
```

cam68 在帧 63、70、71、74 出现了错误选择，全部由历史状态污染导致。

### 1.2 根因分析

问题不在于滤波算法本身（线性外推、Kalman 都试过），而在于**架构设计**：

```
❌ 错误架构（滤波反馈到检测）：

    滤波器状态 ──→ 预测位置 ──→ 影响blob选择 ──→ 选错blob ──→ 污染滤波器状态
         ↑                                                          │
         └──────────────────────────────────────────────────────────┘
                           反馈回路（恶性循环）
```

```
✅ 正确架构（检测与滤波解耦）：

    阶段1: 逐帧独立检测 ──→ 原始检测序列 [d₁, d₂, ..., dₙ]
                                    │
    阶段2: 后处理滤波    ←──────────┘
                                    │
                              平滑轨迹 [s₁, s₂, ..., sₙ]
```

**关键区别**：阶段2的输出不会反馈到阶段1，因此即使滤波出错，也不会影响原始检测。

### 1.3 同类项目的经验

| 领域 | 项目/系统 | 做法 |
|------|----------|------|
| 板球追踪 | Hawk-Eye | 检测完全独立，轨迹用 Kalman 后处理平滑 |
| 细胞追踪 | Cell Tracking Challenge | 先检测所有细胞，再用匈牙利算法做帧间关联 |
| 动作捕捉 | Vicon/OptiTrack | 标记点独立检测，缺失帧用样条插值补 |
| 粒子追踪 | PIV | 先找所有粒子，再用互相关做轨迹链接 |
| 自动驾驶 | 多目标追踪 | 检测器与追踪器(SORT/DeepSORT)严格分离 |

**共同点**：检测（detection）和关联/滤波（association/filtering）是两个独立阶段。

## 2. 当前实现：纯无状态检测

移除 BallTracker 状态后的 pipeline：

```
原始帧 → TrackNet → heatmap → OSD mask → threshold → 连通域 → 加权质心
    → 多blob候选 → court-X 过滤 → top-1 by blob_sum → 输出
```

每一帧的检测完全独立，不受前后帧影响。

**评估结果（33帧 GT 对比）：**

| 指标 | cam68 | cam66 |
|------|-------|-------|
| 正确率 | 97.0% (32/33) | 84.8% (28/33) |
| 错误选择 | 0 | 0 |
| 漏检 | 1 | 5 |
| 平均像素误差 | 2.6px | 3.3px |

错误选择为 0，说明无状态检测在 blob 选择上是可靠的。
漏检帧均为 heatmap 信号在阈值下过弱（球在画面边缘/顶部）。

## 3. 候选后处理滤波方法

以下方法均在检测完成后独立运行，不影响检测环节。

### 3.1 中值滤波（Median Filter）

**原理**：滑动窗口内取中值，天然抗单帧跳点。

```python
from scipy.signal import medfilt
x_smooth = medfilt(x_raw, kernel_size=5)
y_smooth = medfilt(y_raw, kernel_size=5)
```

**特点**：
- 单帧异常值（跳点）会被自动忽略
- 不需要调参，非常稳健
- 不会改变轨迹的整体走势
- 对连续多帧丢失无能为力

**适用场景**：去除偶发的单帧检测跳点

**类比**：细胞追踪中常用，用于去除检测抖动

### 3.2 Savitzky-Golay 滤波

**原理**：滑动窗口内做多项式最小二乘拟合，输出拟合值。

```python
from scipy.signal import savgol_filter
x_smooth = savgol_filter(x_raw, window_length=7, polyorder=2)
y_smooth = savgol_filter(y_raw, window_length=7, polyorder=2)
```

**特点**：
- 保留轨迹的真实形态（加速、减速、转弯）
- polyorder=2 刚好匹配网球抛物线轨迹
- window_length 控制平滑程度
- 比简单移动平均更好地保留峰值和拐点

**适用场景**：平滑轨迹同时保持物理真实性

**类比**：PIV 粒子追踪、光谱信号处理中的标准方法

### 3.3 前向 Kalman 滤波（后处理模式）

**原理**：用状态空间模型（位置+速度）做递推估计，但仅用于平滑输出。

```python
# 状态: [x, y, vx, vy]
# 观测: [x, y]

for frame_idx in range(total_frames):
    kalman.predict()

    if detections[frame_idx] is not None:
        kalman.correct(detections[frame_idx])   # 有检测则修正

    smoothed[frame_idx] = kalman.state[:2]       # 输出平滑位置

    # 关键：smoothed 不反馈给检测，也不影响下一帧的 blob 选择
```

**特点**：
- 能处理丢帧（predict-only 步骤自动外推）
- 自适应置信度（检测噪声大时更依赖预测，反之更依赖观测）
- 需要调参（过程噪声 Q、观测噪声 R）
- 对匀速运动效果好，急变向时滞后

**适用场景**：轨迹平滑 + 短期丢帧补全

**类比**：Hawk-Eye 系统的轨迹重建

### 3.4 样条插值（Spline Interpolation）

**原理**：用有检测的帧作为控制点，拟合平滑曲线，自动补全缺失帧。

```python
from scipy.interpolate import CubicSpline

detected_frames = [51, 52, 53, 55, 56, ...]   # 跳过了 54
detected_x = [838.7, 853.4, 861.4, 868.8, ...]

cs_x = CubicSpline(detected_frames, detected_x)
all_x = cs_x(range(51, 84))   # 自动补全第 54 帧
```

**特点**：
- 自然地填补丢失帧
- 曲线平滑且连续（C2连续）
- 需要足够多的有效检测点（至少4个）
- 在长连续丢帧段可能产生不合理的外推

**适用场景**：补全短期丢帧（1-3帧）

**类比**：动作捕捉系统（Vicon）处理标记点遮挡的标准做法

### 3.5 速度阈值剔除

**原理**：检测帧间速度，超过物理极限的跳变判定为异常。

```python
MAX_SPEED = 200  # 像素/帧，根据球速和帧率估算

for i in range(1, len(positions)):
    speed = np.linalg.norm(positions[i] - positions[i-1])
    if speed > MAX_SPEED:
        positions[i] = None  # 标记为异常，后续用插值补
```

**特点**：
- 利用物理约束（网球最高速度有限）
- 简单有效，无需复杂模型
- 需要根据相机视角和帧率设定合理阈值
- 球被击打瞬间速度突变可能被误判

**适用场景**：作为其他滤波方法的前置步骤，先剔除明显异常值

## 4. 推荐组合方案

```
逐帧独立检测（当前 pipeline，不改）
    │
    ▼
① 速度阈值剔除 ──→ 去掉物理上不可能的跳点
    │
    ▼
② 中值滤波 ──→ 去掉小幅抖动/噪声
    │
    ▼
③ 样条插值 ──→ 填补丢失帧（1-3帧短缺口）
    │
    ▼
④ Savitzky-Golay ──→ 最终平滑，保留轨迹形态
    │
    ▼
  输出：平滑的完整轨迹
```

### 不同应用场景的选择

| 场景 | 推荐方法 | 理由 |
|------|---------|------|
| 实时展示 3D map | 中值滤波（窗口3） | 延迟小，去抖动效果够 |
| 回放/分析 | 全套①②③④ | 质量优先，不在乎延迟 |
| 只需去跳点 | 速度阈值 + 中值 | 最简单 |
| 需要补丢帧 | 样条插值 | 连续轨迹 |

## 5. 参数参考

### 网球物理约束

| 参数 | 值 | 说明 |
|------|-----|------|
| 最高球速 | ~250 km/h (~70 m/s) | 发球极限 |
| 帧率 | 25 fps | 当前摄像头 |
| 每帧最大位移 | ~2.8 m/帧 (70/25) | 世界坐标 |
| 像素位移 | 视角相关 | 需根据单应性估算 |
| 中值滤波窗口 | 3-5 | 实时用3，离线用5 |
| SG 窗口 | 5-9 | 根据轨迹段长度 |
| SG 阶数 | 2 | 匹配抛物线轨迹 |
| 样条最少控制点 | 4 | CubicSpline 的最低要求 |

### 需要注意的边界情况

1. **球被击打瞬间**：速度和方向突变，滤波可能过度平滑拐点
2. **球弹地瞬间**：速度方向反转，低阶滤波可能产生伪影
3. **长连续丢帧（>5帧）**：样条外推不可靠，应标记为"不确定"
4. **多段轨迹**：死球期间应分段处理，不跨段滤波
